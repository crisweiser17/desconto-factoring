Excelente ideia! Migrar para o ecossistema moderno de frontend vai transformar completamente a experiência do sistema, deixando-o muito mais leve e fluido.

Além do **Tailwind CSS** e **Alpine.js**, incluí outras práticas modernas de arquitetura que vão elevar o nível do seu software. Aqui está o plano de refatoração geral atualizado:

### 1. Modernização do Frontend (UX/UI)
- **Migração para Tailwind CSS:**
  - Substituir o Bootstrap 5 pelo Tailwind CSS (via CDN inicialmente, ou via Build/Vite se preferir).
  - Isso permitirá designs muito mais customizados, limpos (estilo "SaaS moderno") e tabelas altamente responsivas.
- **Interatividade com Alpine.js:**
  - Substituir todo o JavaScript puro ("Vanilla JS" / manipulação manual de DOM no `index.php`) por Alpine.js (`x-data`, `x-model`).
  - O estado do formulário da calculadora ficará reativo: ao mudar a taxa, a tela inteira se recalcula e se atualiza instantaneamente sem precisar escrever funções JS complexas de `document.getElementById`.
- **Máscaras de Input Modernas:**
  - Integrar uma biblioteca moderna como Alpine.js Mask ou Cleave.js para garantir que campos de moeda (R$) e CPF/CNPJ sejam formatados suavemente enquanto o usuário digita.

### 2. Segurança e Autenticação (Crítico)
- **Migrar Senha para Banco de Dados:**
  - Criar tabela `usuarios` (`id`, `email`, `senha_hash`).
  - Remover a senha "chumbada" (`Qazwsx123@`) do arquivo `processa_login.php`.
  - Usar `password_hash()` e `password_verify()` nativos do PHP.
- **Proteção CSRF:**
  - Implementar tokens Anti-CSRF em todos os formulários (Login, Configurações, Deletar dados) para evitar requisições forjadas.

### 3. Melhorias Estruturais (Backend e Banco)
- **Soft Delete (Exclusão Lógica):**
  - Em sistemas financeiros, nada deve ser deletado fisicamente. Adicionar coluna `deleted_at` em `cedentes` e `sacados` para que, ao excluir, eles sumam da tela, mas os relatórios e históricos passados nunca quebrem.
- **Separação de Responsabilidades (MVC Light):**
  - O `index.php` tem mais de 1400 linhas. Vamos separar em:
    - `views/` (O HTML com Alpine/Tailwind)
    - `controllers/` (Onde fica a lógica do PHP)
    - `assets/` (Imagens e lógicas genéricas)
- **Ocultar Erros Críticos:**
  - Em arquivos como `registrar_operacao.php`, os erros do PDO (`$e->getMessage()`) estão indo direto para o navegador. Vamos gravar isso em log (`error_log`) e retornar uma mensagem "limpa" para o usuário final, evitando expor a estrutura do banco para hackers.

### 4. Geração de PDF Moderna
- **Refatorar o FPDF:**
  - Atualmente o FPDF gera os relatórios de forma muito estática. Podemos atualizar o layout do PDF para combinar com a nova interface do Tailwind (cores, fontes modernas e logos mais bem posicionados).

Como é uma mudança estrutural grande, sugiro atacarmos em Fases para que o sistema nunca fique quebrado:
- **Fase 1:** Resolver a Segurança (Tabela de usuários e remoção da senha no código).
