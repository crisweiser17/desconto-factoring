O sistema está utilizando o banco de dados **MySQL** (ou MariaDB), acessado através da extensão PDO do PHP, conforme verificado no arquivo `db_connection.php` e pelo formato do dump `banco_dump.sql.s`.

Para criar um instalador amigável e dinâmico, proponho o seguinte plano:

## Plano de Implementação: Instalador Web (setup.php / installer.php)

### 1. Interface de Configuração (Frontend)
- Substituir o atual `installer.php` (que possui credenciais fixas) por uma interface interativa usando Bootstrap (já presente no projeto).
- Criar um formulário solicitando as seguintes informações:
  - **Servidor (Host)** (ex: localhost)
  - **Usuário do Banco** (ex: root)
  - **Senha do Banco**
  - **Nome do Banco de Dados** (ex: factor_db)

### 2. Processamento e Conexão (Backend)
- Ao submeter o formulário, o script tentará conectar ao servidor MySQL usando as credenciais informadas via PDO.
- Validar a conexão com tratamento de erros amigável (try/catch), exibindo alertas claros caso a senha ou usuário estejam incorretos.

### 3. Criação do Banco e Estrutura
- O script criará o banco de dados especificado (caso não exista).
- Em seguida, fará a leitura do arquivo de dump existente (`banco_dump.sql.s`) e o dividirá em comandos SQL individuais.
- Executará os comandos para construir toda a estrutura de tabelas (`cedentes`, `sacados`, `operacoes`, etc.) e inserir os dados básicos.

### 4. Geração Automática do Arquivo de Conexão
- O instalador irá gerar/sobrescrever o arquivo `db_connection.php` dinamicamente com as variáveis fornecidas pelo usuário no formulário, para que o sistema já fique 100% funcional.

### 5. Finalização e Segurança
- Exibir tela de sucesso com links rápidos para acessar o sistema.
- Adicionar um aviso de segurança orientando a remoção do script de instalação após o uso, para evitar que terceiros resetem o banco de dados.
